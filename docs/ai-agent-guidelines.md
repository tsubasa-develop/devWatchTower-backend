# AI Agent Development Guidelines

このリポジトリは AI エージェントによる自動実装も想定しています。

## コメントについて

- 原則としてコメントは不要です（冗長な説明コメントは追加しない）。
- コメントで意図を補うのではなく、命名・型・関数分割で意図が伝わる実装にする。
- 例外として、コメントがないと誤解や事故につながる場合のみ最小限許可します。
  - 外部仕様に依存する制約（API 仕様、DB の制約、互換性のための特殊事情）
  - セキュリティ上の理由（なぜその検証が必要か等）
  - 将来の削除が前提の暫定対応（期限/条件が明確なもの）

## 追加ルール

- 変更内容は PR / Issue の本文にまとめ、コード内には極力残さない。
- 既存コードに冗長なコメントがある場合、機能に影響しない範囲で削除してよい。
